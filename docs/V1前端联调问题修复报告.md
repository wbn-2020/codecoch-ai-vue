# V1 前端联调问题修复报告

## 1. 本轮前端修复范围

本轮只处理 `CodeCoachAI-vue` 前端 P0 问题，未修改后端、数据库或新增接口。

修复范围：

- 题目管理页展示与提交字段收敛。
- 分类管理页 V1 字段精简、本地筛选、本地分页。
- 标签管理页 V1 字段精简、本地筛选、本地分页。
- 问题组管理页本地筛选、本地分页，并隐藏 V1 暂不可靠的题目数。
- Prompt 模板页查询参数改为后端已支持的 `scene`，隐藏变量说明。
- 面试房间补齐未开始、无当前题、开始后刷新当前题的状态处理。
- 面试报告页补充总分来源说明和空状态。

- 开始后仍没有题目时显示明确提示。
- 未调用旧 `/ai/interview/**` 接口。

### 面试报告页

- 新增总分来源说明：总分来自后端 `totalScore` 字段，前端不重新计算。
- 阶段得分为空时显示空状态，不渲染空图表。
- 其他文本字段为空时继续显示“暂无...”类空内容。
- 未展示学习计划、AI 简历优化等 V2 功能。

## 6. 静态扫描结果

### inner / 旧用户端 AI 面试接口

命令：

```powershell
rg -n '/inner|inner/|questions/answer|questions/wrongs|/ai/interview/question|/ai/interview/evaluate|/ai/interview/follow-up|/ai/interview/report' src
```

结果：无命中。

结论：没有 `/inner/**`，没有旧 `/ai/interview/**`，没有旧 `questions/answer` 或 `questions/wrongs`。

### 分类 / 标签 / 问题组 status 子路径

命令：

```powershell
rg -n '/admin/question-categories/.*/status|/admin/question-tags/.*/status|/admin/question-groups/.*/status' src
```

结果：无命中。

结论：分类、标签、问题组不再调用不存在的 `/status` 子路径。

### questionType / questionGroupId / tagIds

命令：

```powershell
rg -n 'questionType|questionGroupId|tagIds' src
```

结果：仍有命中。

合理保留：

- `questionType`：用户端题库和详情页仍用于展示后端返回的题型信息，`QuestionMeta` 和类型定义保留兼容；管理员题目管理页已隐藏题型列和题型表单。
- `questionGroupId`：仅在 `src/api/interview.ts` 中兼容后端返回的当前问题字段，不是创建面试提交字段。
- `tagIds`：管理员题目维护仍需要多标签保存；用户端 `/questions` 查询已通过 `tagId` 单选，并在 API 层置空 `tagIds` 防止误传。

结论：剩余命中不是本轮 P0 阻塞项。
