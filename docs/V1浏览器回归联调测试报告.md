- 管理端 `/admin/questions` 支持标签和状态筛选。
- 题目管理题型列和题型表单项已隐藏。
- 题目编辑保存链路在补全必填字段后正常，保存 payload 不包含 `questionType`。

- 问题组管理隐藏题目数列，并支持关键词、分类、状态、本地分页、新增、编辑、删除。
- Prompt 模板支持 keyword、scene、status 筛选，变量说明列已隐藏，编辑保存正常。
- 用户端创建面试不展示、不提交 `questionGroupId`，不调用 `/admin/question-groups`。
- 面试房间开始前禁用回答，开始后可获取题目并提交回答。
- 面试报告总分来源说明和阶段得分空状态已显示。

## 10. 下一步建议

1. 先修复用户题库标签筛选的数据契约：后端用户题库列表返回真实 `tagIds` 或结构化 `tags[{id,name}]`，避免前端 hash 名称生成伪 id。
2. 再修复面试报告生成/Mock AI 数据：报告 summary、strengths、weaknesses、suggestions 必须输出中文，并去除 `Mock report` 字样。
3. 修复后只重测用户端题库标签筛选和面试报告，不扩大到 V2/V3 或新增接口。
