# V1 前端剩余 P0 阻塞问题修复报告

## 1. 本轮修复范围

本轮只修用户端题库标签筛选数据契约，不修改后端、不新增接口、不新增 mock 数据、不扩展 V2/V3 功能。

涉及文件：

- `src/api/question.ts`
- `src/views/question/QuestionListView.vue`
- `src/components/question/QuestionMeta.vue`
- `src/views/question/QuestionDetailView.vue`
- `src/types/question.ts`

## 2. 修复说明

### 标签 ID 归一化

- 移除标签名称 hash 生成伪 ID 的逻辑。
- 标签归一化优先使用后端返回的 `tags[].id`。
- 当 `tags[].id` 缺失时，兼容使用同位置的 `tagIds[index]`。
- 当旧数据只有 `string[]` 标签且没有 `tagIds` 时，仅保留标签名称用于展示，`id` 置为 `0`，不进入筛选项。
- 兼容后端新增的 `tagNames` 字段。

### 用户端题库筛选项

- `QuestionListView.vue` 生成标签筛选项时只保留真实 `tag.id > 0` 的标签。
- 展示名称优先使用 `tag.name`，其次使用 `tag.tagName`。
- 筛选项按真实 `tag.id` 去重，避免同名但不同 ID 的标签被错误合并。

### 查询参数

- `getQuestionsApi` 通过解构移除 `tagIds`，只传 `tagId` 单选参数。
- `getFavoriteQuestionsApi` 同步移除 `tagIds`，避免用户端收藏题列表误传多标签参数。

### 标签展示兼容

- `QuestionMeta.vue` 展示标签时兼容 `tag.name || tag.tagName`。
- `QuestionDetailView.vue` 兼容旧 `string[]` 和新对象数组结构，不再为旧字符串标签生成伪 ID。

## 4. 回归结果

### 后端标签筛选契约

使用普通用户 `e2e_user / E2eUser@123` 登录后调用：

```text
GET http://localhost:8080/questions?pageNo=1&pageSize=20&tagId=100002
```

结果：

- 请求参数使用 `tagId=100002`。
- 返回 `total=3`。
- 返回题目包含真实标签 `id=100002`，名称为 `E2E_TEST_缓存`。
- 未使用 `tagIds` 参数。

### 用户端 admin 接口边界

扫描范围：

```powershell
rg -n '/admin/question-tags|/admin/question-categories' src/views/question src/components/question src/api/question.ts
```

结果：无命中。

结论：用户端题库、题目详情、题库筛选组件和用户端题库 API 未调用 `/admin/question-tags` 或 `/admin/question-categories`。

### tagIds 误传检查

`src/api/question.ts` 中用户端查询已通过解构移除 `tagIds`：

- `getQuestionsApi`
- `getFavoriteQuestionsApi`

剩余 `tagIds` 命中用于兼容后端响应或管理员题目维护，不属于用户端 `/questions` 查询误传。
